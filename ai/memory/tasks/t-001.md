## What you're building

## Watch out for (Node â†’ Java)

- **DI is runtime container wiring, not `import`.** You don't construct or import `TaskService`; you declare beans and Spring injects them at startup. Use constructor injection so dependencies are explicit and testable â€” avoid field/`@Autowired` injection.
- **A loaded JPA entity is "managed."** Inside a transaction, mutating a fetched entity persists on flush with no explicit `save()` call. (Not central here since you mostly create-and-read, but it's the mental model that will bite you later.)
- **Use narrow test slices, not `@SpringBootTest`.** `@DataJpaTest` loads only the persistence layer; `@WebMvcTest` loads only the web layer with a mocked service. Reach for the slice that matches what you're testing â€” full-context tests are slow and blunt.
- **It's `jakarta.*`, not `javax.*`.** Any tutorial using `javax.persistence` is pre-2023 and will not compile on Boot 4.
